      # 开发说明

      # 结构

      ----docs    //组件库文档
-----build   // 打包组件库的webpack配置
-----lib     // 打包后的组件库（npm包）
-----src     //vue开发环境
-------lib  //创建组件库的地方。

      开发组件库和书写组件库文档时应严格按照目录结构来。否则会造成打包错误，打包失败的等奇葩问题。

      # css

      推荐使用BEM命令的方式，避免出现层叠的css，便于外部修改。同时为了避免和其他ui库的css冲突建议在zy-为开头例如：

      <div class="zy-body">
  <h1 class="zy-body__h1"></h1>
 ......
</div>

      关于更多BEM的详细信息可以看：

      • 快速了解BEM
      • BEM官网

      --src
---lib
----components // 组件包
-----example  //单个独立组件
------index.js //注册组件到vue的地方。
------index.vue //组件文件
----base.css //全局/通用css

      # docs组件文档

      ---docs
----.vuepress // 配饰vuepress的地方，配置路由，侧边栏。顶部栏等等。
-----enhanceApp.js //全局引入写好的组件库以便在md文档中直接使用和展示写好的组件库
-----sidebars.js //配置侧边栏
----mobile //手机端组件文档
----pc //pc端组件文档

      书写文档框架主要是用vuepress，使用markdown来书写文档。

      点击了解更多关于vuepress

      # .vue文件的name

      当你开发一个组件时，比如你开发的是一个Input输入组件在.vue组件的name里必须带上Zy前缀，以首字母大写驼峰大小写的方式。避免和其他组件库命名冲突

      <template>
  <div class="zy-input">
   .....
  </div>
</template>

<script>
export default {
  name: "ZyExample",
  .....
}
